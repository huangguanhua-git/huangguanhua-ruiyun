# 自定义操作系统构建与部署CI/CD流水线说明

## 概述
本GitLab CI/CD流水线用于自动化构建自定义Ubuntu操作系统镜像，并将其部署到MAAS环境中。流水线包含5个阶段，每个阶段可自定义手动或自动触发。

## 流水线阶段
**1. 制作ISO镜像(`1_make_iso`)**  
**功能**: 清理环境并构建基础ISO镜像  
**主要操作**: 清理之前的构建进程(mksquashfs,xorriso,make,packer)；移除LXD snap环境；复制构建数据并执行构建脚本  
**触发方式**: 手动或自动触发

**2. 制作Packer压缩包(`2_make_packer_tar`)**  
**功能**: 使用Packer制作自定义Ubuntu压缩包  
**主要操作**: 清理环境进程；修改Packer配置；启动后台构建进程并记录PID  
**产出物**: `make.pid`(构建进程ID)  
**触发方式**: 手动或自动触发

**3. 加速压缩(`3_accelerated_compression`)**  
**功能**: 等待镜像生成并执行加速压缩  
**主要操作**: 监控压缩进程状态；执行加速压缩脚本  
**触发方式**: 手动或自动触发

**4. 上传压缩包(`4_upload_tar`)**  
**功能**: 上传构建的镜像到MAAS服务器  
**主要操作**: 清理所有构建进程；上传镜像文件到MAAS；在MAAS中创建启动资源  
**触发方式**: 手动或自动触发

**5. 部署操作系统(`5_deploy_os`)**  
**功能**: 在目标机器上部署自定义操作系统  
**主要操作**: 安装必要的工具(sshpass)；通过跳板机连接到目标机器(10.1.5.4)；执行系统部署  
**触发方式**: 手动或自动触发

## 使用说明
**前提条件**  
GitLab Runner已配置并标记为`custom-runner-1`；具有足够的权限执行sudo命令；网络可访问MAAS服务器和目标机器

**执行顺序**  
按顺序执行每个阶段：1→2→3→4→5；每个阶段完成后，当前模式为自动,可修改为手动方便调试；如需改为自动把manual配置去除，如果某个阶段失败，需要排查问题后重新执行

**网络配置**  
**runner主机**：10.10.249.79   
**跳板机**: 10.10.249.214  
**目标机器**: 10.1.5.4  
**认证方式**: 密钥登录

**重要文件**  
`all.sh`-主要构建脚本；`make_hcl.sh`-Packer配置脚本；`wait_for_compression.sh`-压缩监控脚本；`cp.yaml`-镜像上传Ansible Playbook；`deploy.yaml`-系统部署Ansible Playbook；`maasip`-目标主机清单文件 **update-gitci.sh** 为提交代码脚本,执行后需要输入更新说明，如不输入，默认提交说明为当前时间。
## 故障排除
**常见问题**  
权限不足:确保Runner用户有sudo权限；进程冲突:每个阶段开始时会自动清理之前的进程；网络连接:确保可以访问跳板机和目标机器；构建超时:监控构建日志，必要时调整超时设置

**日志文件**  
`/mnt/custom-os/xxx架构/make.log`-主要构建日志；`/mnt/custom-os/xxx架构/time.log`-压缩时间日志

## 维护说明
定期检查构建环境是否正常；更新操作系统基础镜像时可能需要调整构建脚本；确保MAAS API访问权限有效

**注意**: 此流水线包含敏感信息，请妥善保管相关配置文件
